# Product Guidelines: Apicurio Registry Terraform Provider

## Prose Style
- **HashiCorp Standard:** All documentation must follow the standard HashiCorp documentation format, ensuring clear, consistent, and structured resource descriptions.

## Error Messages
- **User-Friendly Tone:** Error messages should provide user-friendly guidance and actionable steps. Whenever possible, include links or references to relevant documentation to help users quickly resolve the issue.

## Code Examples
- **Fully Functional Configuration:** Example code in the documentation must be complete, runnable Terraform configurations to ensure users can copy, paste, and execute them with minimal modification.
