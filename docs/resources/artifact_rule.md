# apicurio_artifact_rule (Resource)

Manages a rule applied to an artifact in [Apicurio Registry](https://www.apicur.io/registry/). Rules control schema validation and compatibility enforcement when new versions of an artifact are registered.

Two rule types are supported:

- **VALIDITY** â€” Validates that new artifact content is syntactically correct.
- **COMPATIBILITY** â€” Ensures new versions are compatible with previous versions according to the configured strategy.

## Example Usage

### Backward Compatibility Rule

```terraform
resource "apicurio_artifact_rule" "backward_compat" {
  group_id    = "com.example"
  artifact_id = "user-schema"
  type        = "COMPATIBILITY"
  config      = "BACKWARD"
}
```

### Full Compatibility Rule

```terraform
resource "apicurio_artifact_rule" "full_compat" {
  group_id    = "com.example"
  artifact_id = "user-schema"
  type        = "COMPATIBILITY"
  config      = "FULL"
}
```

### Syntax Validity Rule

```terraform
resource "apicurio_artifact_rule" "syntax_check" {
  group_id    = "com.example"
  artifact_id = "user-schema"
  type        = "VALIDITY"
  config      = "SYNTAX_ONLY"
}
```

### Combined with Artifact Resource

```terraform
resource "apicurio_artifact" "payment_schema" {
  group_id    = "com.example"
  artifact_id = "payment-schema"
  type        = "AVRO"
  content     = file("${path.module}/schemas/payment.avsc")
}

resource "apicurio_artifact_rule" "payment_compat" {
  group_id    = apicurio_artifact.payment_schema.group_id
  artifact_id = apicurio_artifact.payment_schema.artifact_id
  type        = "COMPATIBILITY"
  config      = "BACKWARD_TRANSITIVE"
}

resource "apicurio_artifact_rule" "payment_validity" {
  group_id    = apicurio_artifact.payment_schema.group_id
  artifact_id = apicurio_artifact.payment_schema.artifact_id
  type        = "VALIDITY"
  config      = "SYNTAX_ONLY"
}
```

## Schema

### Required

- `artifact_id` (String) The ID of the artifact to apply the rule to.
- `config` (String) The rule configuration value. Valid values depend on the rule `type`:
  - For `COMPATIBILITY`: `BACKWARD`, `BACKWARD_TRANSITIVE`, `FORWARD`, `FORWARD_TRANSITIVE`, `FULL`, `FULL_TRANSITIVE`, `NONE`
  - For `VALIDITY`: `SYNTAX_ONLY`, `NONE`
- `type` (String) The rule type. Must be one of: `COMPATIBILITY`, `VALIDITY`.

### Optional

- `group_id` (String) The group ID of the artifact. Defaults to `default`.

### Read-Only

- `id` (String) The resource ID in format `group_id/artifact_id/type`.

## Compatibility Config Reference

| Config | Description |
|--------|-------------|
| `BACKWARD` | New version can be used by consumers using the previous version |
| `BACKWARD_TRANSITIVE` | New version is backward compatible with all previous versions |
| `FORWARD` | Previous version can be used by consumers using the new version |
| `FORWARD_TRANSITIVE` | Previous versions can be used by consumers using the new version |
| `FULL` | Both backward and forward compatible with the previous version |
| `FULL_TRANSITIVE` | Both backward and forward compatible with all previous versions |
| `NONE` | No compatibility checking |

## Validity Config Reference

| Config | Description |
|--------|-------------|
| `SYNTAX_ONLY` | Only validate that the schema is syntactically correct |
| `NONE` | No validity checking |

## Import

Artifact rules can be imported using the format `group_id/artifact_id/type`:

```shell
terraform import apicurio_artifact_rule.example "com.example/user-schema/COMPATIBILITY"
```

Import using the default group:

```shell
terraform import apicurio_artifact_rule.example "default/my-artifact/VALIDITY"
```
